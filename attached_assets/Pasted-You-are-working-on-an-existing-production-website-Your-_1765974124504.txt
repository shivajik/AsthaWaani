You are working on an existing production website.
Your task is to ADD a Simple CMS without breaking any existing functionality, layout, SEO, or URLs.

IMPORTANT RULES:
- Do NOT refactor existing pages unless required
- Do NOT change existing public URLs
- CMS must be lightweight, fast, and secure
- Existing website must continue to work exactly the same
- Follow SEO, AEO, and LLM SEO best practices
- Use clean, readable, production-ready code

--------------------------------
CMS SCOPE (Simple, Essential Only)
--------------------------------

1. Authentication (Admin Only)
- Create a secure admin login (/admin)
- Email + password based login
- Passwords must be hashed
- Protect all CMS routes with auth middleware
- No public user registration

2. Admin Dashboard
- Clean, minimal dashboard UI
- Sidebar with:
  - Pages
  - Blog Posts
  - Media
  - SEO Settings
  - Settings
  - Logout

3. Page Management
- View all existing pages
- Edit page content:
  - Page title
  - Meta title
  - Meta description
  - Slug (read-only for existing pages)
  - Main content (rich text editor)
- Save changes and reflect instantly on live site
- Do NOT change page URLs

4. Blog Management
- Create, edit, delete blog posts
- Fields:
  - Title
  - Slug (auto-generated, editable before publish)
  - Featured image
  - Content (rich text)
  - Meta title
  - Meta description
  - Schema (Article)
- Draft & Publish status
- SEO-friendly URLs

5. Media Manager
- Upload images
- Auto-optimize images (size & format)
- Store alt text for every image
- Prevent duplicate uploads
- Reuse images across pages/posts

6. SEO Controls (Very Important)
- Editable meta title & description per page/post
- Canonical URL field
- OpenGraph + Twitter tags
- Auto-generate schema:
  - Organization
  - LocalBusiness
  - Article
- Ensure clean HTML output (no bloated divs)

7. Database
- Use existing DB if present, otherwise create:
  - admins
  - pages
  - posts
  - media
  - seo_meta
- Follow proper relations
- No redundant tables

8. Frontend Integration
- Public pages must read content from CMS
- Content should render server-side (SEO friendly)
- No client-only rendering for main content
- Ensure fast load time

9. Security
- CSRF protection
- Input sanitization
- XSS protection
- File upload validation
- Rate limit admin login

10. Performance
- Cache public pages
- Lazy load images
- No heavy libraries unless necessary

--------------------------------
TECH & UX PREFERENCES
--------------------------------
- Simple UI (no overengineering)
- Mobile responsive admin panel
- Use a proven rich text editor (lightweight)
- Follow MVC or clean architecture
- Comment the code where logic is important

--------------------------------
FINAL STEP
--------------------------------
After implementation:
- Verify existing pages still load correctly
- Verify SEO tags render properly in page source
- Ensure CMS changes reflect immediately
- Provide brief instructions on how to use CMS

Start by analyzing the existing codebase,
then implement ONLY what is missing.
